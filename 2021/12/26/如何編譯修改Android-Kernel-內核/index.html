如何編譯修改Android Kernel(內核)

發表於 2021-12-26 更新於 2022-01-14 分類於 電腦科學 Disqus：

本文簡述如何自行修改Android kernel，開啟特定的功能，再重新編譯檔案，將其刷入手機。
先從簡單的開始: 刷入自己修改過的kernel後，會在手機核心版本顯示自己的名字。

如果成功了，那麼就能更進一步去調kernel的功能，例如把docker要求的功能全部打開，就能在Android手機跑docker。

一、硬體需求

要刷kernel，手機必定已經解鎖。刷kernel不會重置手機資料，所以刷之前只要備份boot分區，避免卡開機。

本文使用的手機是紅米Note 5 (whyred)，系統為LineageOS 18。

編譯kernel需要一台x86-64位元的Linux電腦。我的電腦規格: Intel Core 2 Q9550 + 4GB DDR2 RAM，作業系統: Lubuntu 20.04。

二、步驟概述

編譯kernel包含除錯可能會花至少一天的時間。

下載kernel原始碼 → 下載交叉編譯器 → 簡單加上名字後第一次編譯 → 刷入手機看功能正不正常 → menuconfig修改kernel → 重新編譯 → 刷入到手機

三、取得kernel原始碼

原廠的kernel或第三方kernel都可以，通常會把原始碼放在Github。
這裡我以有很多額外功能的”RAD Kernel”專案做示範，這個專案程式碼問題比較少。

  1. 到他們的Github，使用指令clone原始碼
  2. 為了方便識別，將clone下來的目錄重新命名為”src”。

四、下載交叉編譯器

  1. 這裡使用EVA GCC編譯。到XDA下載ARM與ARM64的版本(點選direct download下載)

  2. 將下載的arm版本解壓縮，命名為tc32。

  3. aarch64解壓縮，命名為tc。

  4. 將這二個目錄移動到customkernel，到目前為止目錄長這樣，src是kerenl原始碼的目錄，tc和tc32則是剛剛下載的交叉編譯器。

五、第一次測試編譯

  1. 進入src的目錄，在這裡開啟終端機。先export環境變數:

  2. 按照機型_config檔案產生設定，該檔案通常在arch/arm64/configs。

  3. 開啟src目錄的MakeFile，在EXTRAVERSION後面加入自己名字的字串:

  4. 接著開始編譯kernel

  5. 這台電腦編譯至少要20分鐘，若遇到error編譯器就會停下來，要去改程式碼再重新make，編譯器會從上個中止的地方繼續。關於常見錯誤，參考這篇: 編譯Android kernel遇到的error紀錄

  6. 編譯好的檔案位於arch/arm64/boot/，應該會有一個Image.gz-dtb的檔案。

  7. 接著要重新打包boot.img，把原廠的boot.img解開之後把我們做的kernel塞進去。到XDA下載Linux版AIK (點選文中的AIK-Linux-v3.8-ALL.tar.gz附件)，解壓縮。

  8. 手機進入TWRP → Advanced → Terminal，使用dd指令從手機提取原廠的boot.img。

  9. 把這個原廠的stockboot.img檔案傳輸到電腦，放到AIK的工作目錄，並將剛剛編譯的Image.gz-dtb也放到這個目錄。

  10. 在AIK目錄開啟終端機，使用指令解開stockboot.img(需要sudo)

  11. 進入目錄split_img，把stockboot.img-kernel檔案替換成我們編譯好的Image.gz-dtb(檔名要改成stockboot.img-kernel)。

  12. 回到AIK目錄，重新打包，應該會得到一個image-new.img的檔案。

六、刷入到手機

將新的image-new.img傳輸到手機，用TWRP點選Install Image，刷入到boot分區。

接著就是看看能不能開機了…若卡開機，還原TWRP的boot分區備份，繼續在電腦上debug吧…

七、menuconfig開關kernel功能

如果前六個步驟成功完成，那就來真正修改kernel吧。

  1. 在src目錄開啟終端機，輸入指令開啟kconfig選單，用於開關kernel的功能。

  2. 使用鍵盤上下和Enter進入子選項。找到要開啟的項目按下y，然後用鍵盤左右鍵移動到Save儲存設定值。

  3. 改好後切換到Exit退出，寫入變更，再次開始編譯kernel:

  4. 接著重複第五步的步驟，編譯成功後把kernel打包，刷到手機測試。
